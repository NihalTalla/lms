# Bug Fix Summary - Parser Issues

## Issues Found and Fixed

### 1. DEDENT Handling in statement() - FIXED
**Problem**: When `statement()` encountered a DEDENT, it was advancing past it, causing issues in nested structures.

**Fix**: Changed to advance past DEDENT and return null, letting the caller (function/class body loops) handle DEDENT checking.

### 2. Function Body Parsing Loop - FIXED  
**Problem**: The loop wasn't properly handling cases where `statement()` returns null.

**Fix**: Added proper checks for DEDENT and EOF after `statement()` returns null.

### 3. Return Outside Function Error - IN PROGRESS
**Problem**: After parsing nested function `inner`, the parser encounters "return inner" when `inFunction` is false.

**Root Cause**: After parsing nested function, we're seeing:
- DEDENT (ends inner function) ✓
- DEDENT (ends outer function) ✗ - but "return inner" hasn't been parsed yet!
- INDENT (unexpected)
- RETURN (this is "return inner" that should be inside outer function)

**Analysis**: The outer function body parsing is exiting too early, before parsing "return inner". This suggests the nested function parsing is consuming an extra DEDENT or the function body loop is breaking incorrectly.

**Next Steps**: 
1. Check if nested function definition parsing is consuming DEDENT incorrectly
2. Verify function body loop doesn't exit prematurely
3. Check if INDENT token at position 846 is incorrectly generated by lexer

## Current Status
- Parser improvements made but "return outside function" error persists
- Need to investigate nested function parsing more carefully
